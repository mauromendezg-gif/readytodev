Métricas de producto, errores y performance en una sola vista. Sin configuración compleja, sin data engineers. Pensado para equipos de 2 a 15 personas.

Prototipo Publicado: Marzo 2026
Overview del producto

El problema

Las startups en etapa temprana no tienen un data engineer ni tiempo para configurar Datadog, Mixpanel y Sentry por separado. El resultado: decisiones de producto a ciegas, bugs descubiertos por usuarios y performance ignorada hasta que algo explota en producción.

La solución

Un único dashboard que combina errores (tipo Sentry), métricas de producto (tipo Mixpanel) y performance web (Core Web Vitals) — instalable con un script de 1 línea, sin configuración adicional.

Diferenciadores

  • Setup en < 5 minutos: una línea de código, cero configuración
  • Vista unificada: errores + métricas + performance en un solo lugar
  • Alertas inteligentes: solo notifica cuando importa (no spam)
  • Session replay para entender cómo el usuario llegó al error
  • Pricing por startup, no por enterprise (desde $49/mes)
PRD — Requerimientos

Usuarios objetivo

  • CTO / Tech Lead: Quiere visibilidad de errores en producción sin montar infraestructura compleja.
  • Product Manager: Necesita saber qué features usan los usuarios sin exportar CSVs.
  • Founder técnico: Quiere todo en un lugar, paga con tarjeta, no quiere contratos anuales.

Flujos principales

  • Signup → instalar snippet → primer dato llega en < 2 min
  • Dashboard home: resumen de errores nuevos, DAU, p95 de carga
  • Click en error → stack trace + session replay + contexto del usuario
  • Funnel builder drag & drop para definir métricas de conversión
  • Alertas por Slack / email cuando error rate supera umbral
Arquitectura técnica

Ingesta de datos

SDK JavaScript ligero (< 8kb gzipped) que captura errores, eventos de usuario y Web Vitals. Envío en batch cada 5 segundos para no impactar performance del sitio cliente.

Pipeline

Eventos recibidos por Edge Functions (Vercel) → encolados en Vercel Queues → procesados y almacenados en ClickHouse para queries analíticos de alta velocidad.

Stack completo

  • Frontend dashboard: Next.js 15 + Tremor + Recharts
  • SDK cliente: TypeScript vanilla, sin dependencias
  • Almacenamiento: ClickHouse Cloud (eventos) + PostgreSQL (usuarios, proyectos)
  • Session replay: rrweb para grabación del DOM
  • Alertas: Resend (email) + Slack API
